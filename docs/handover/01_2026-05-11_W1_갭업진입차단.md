## 1. 왜 이게 첫 번째인가 — 가설

**가설:**
> 손절 빈발의 1차 원인은 손절폭이 좁아서가 아니라,
> **시초가 갭업(어제 종가 +3% 이상) 자리에서 매수가 일어나기 때문**이다.
> 갭업에서 매수하면 갭 메우기로 정상 변동성에 손절선이 걷어차인다.

**현재 동작:**
1. `08:30 KST` 파이프라인 시작 → 어제 종가 데이터로 분석
2. HITL 자동승인 (모의 모드)
3. `09:00` 장 시작 → `limit_buy_with_timeout` 진입
4. `ENTRY_DISCOUNT = 0.995` (현재가 대비 0.5% 아래)
5. **갭업 시초가에서 사실상 그대로 매수 체결**
6. 종일 갭 메우기로 진입가 -3% 손절 트리거

**기대 효과:**
- 갭업 ≥3% 종목 진입 차단 → 일일 손절률 30% → 15% 수준 기대
- 차단된 자금은 **다음 날 또는 오후 스캔**에 재사용 가능

**파일 (신규):** `autostock/reporting/daily_kpi.py`

매일 17:30 KST에 자동 실행되는 KPI 리포트 작성기. APScheduler에 등록:

```python
async def run_daily_kpi_report() -> None:
    """일일 KPI 텔레그램 리포트."""
    from autostock.db import supabase as db
    from autostock.hitl import telegram_bot as bot_ui

    today = datetime.now(KST).date().isoformat()
    rows = db.fetch_decisions_by_date(today)
    if not rows:
        return

    buy_rows = [r for r in rows if r["action"] == "BUY" and r["order_qty"] > 0]
    blocked = [r for r in rows if r.get("blocked_reason")]

    # 손절 추적
    sold_rows = db.fetch_sells_by_date(today)
    stop_loss_rows = [r for r in sold_rows if r.get("reason") == "손절"]

    avg_gap = (sum(r.get("gap_pct") or 0 for r in buy_rows) / len(buy_rows)) if buy_rows else 0
    avg_slip = (sum(r.get("entry_slippage_pct") or 0 for r in buy_rows) / len(buy_rows)) if buy_rows else 0
    stop_rate = (len(stop_loss_rows) / len(buy_rows) * 100) if buy_rows else 0

    msg = (
        f"📊 *일일 KPI 리포트* — {today}\n\n"
        f"매수: {len(buy_rows)}건 (차단 {len(blocked)}건)\n"
        f"손절률: {stop_rate:.1f}% ({len(stop_loss_rows)}/{len(buy_rows)})\n"
        f"평균 갭업: {avg_gap:+.2f}%\n"
        f"평균 슬리피지: {avg_slip:+.2f}%"
    )
    bot_ui.schedule_message(msg)
```

`scheduler/jobs.py`에 17:30 cron 등록.

> ⚠️ **주의:** 이 리포트는 **W1 첫날부터 작동**해야 합니다. 가드를 걸고 며칠 뒤 측정이 시작되면 비교 기준점(baseline)을 영영 못 잡습니다.

### Step 2. 갭업 진입 차단 가드 (월요일 오후)

**파일:** `autostock/trading/executor.py`

`execute_decisions` 함수의 BUY 분기 진입부에 가드 추가:

```python
# 상단 상수
GAP_UP_BLOCK_PCT = 3.0  # 시초가 갭업 ≥3% 시 신규 진입 보류

# execute_decisions 내, BUY 분기:
if d.action == "BUY":
    # ── 갭업 가드 ──────────────────────────────
    prev_close = w_info.get("prev_close", 0)
    cur_price = 0
    gap_pct = 0
    blocked_reason = None
    try:
        from autostock.trading.kis_client import get_current_price
        cur_price = get_current_price(d.ticker)
    except Exception as e:
        log.warning("[%s] 현재가 조회 실패 — 갭 검증 우회: %s", d.ticker, e)

    if prev_close > 0 and cur_price > 0:
        gap_pct = (cur_price / prev_close - 1) * 100
        if gap_pct >= GAP_UP_BLOCK_PCT:
            log.warning(
                "[%s] 갭업 차단: prev=%.0f cur=%.0f gap=%+.2f%% (>= %.1f%%)",
                d.ticker, prev_close, cur_price, gap_pct, GAP_UP_BLOCK_PCT
            )
            blocked_reason = "GAP_UP"
            qty = 0
            bot_ui.schedule_message(
                f"⏸ *{w_info.get('name', d.ticker)}* 갭업 진입 보류\n"
                f"전일 {prev_close:,.0f} → 시초 {cur_price:,.0f} ({gap_pct:+.1f}%)"
            )

    # ── (기존 매수 로직) ──
    if blocked_reason is None:
        if approved_qty:
            qty = approved_qty.get(d.ticker, 0)
        else:
            qty = calc_order_qty(d, available_cash, position_scale)
```

**검증 포인트:**
- `prev_close == 0` 또는 `cur_price == 0` 인 경우 가드 적용 안 됨 (안전 보수적 동작)
- 차단 시 텔레그램 알림으로 운영자 인지 가능
- `decisions` 테이블에 `blocked_reason` 기록되어 사후 검증 가능
