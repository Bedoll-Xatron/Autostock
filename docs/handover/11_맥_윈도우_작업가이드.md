# 11. 맥 / 윈도우 크로스플랫폼 작업 가이드

**구분:** 상시 참고 문서
**핵심:** **서버 운영 = 윈도우 전용. 맥 = 개발·편집·백테스트용.** (둘을 헷갈리면 사고)

---

## 0. 한 줄 요약

> 코드는 git으로 공유하되, **서버(main.py --serve / PM2)는 윈도우에서만 실행**된다. 맥에서는 클론받아 **편집·백테스트·연구**만 하고, 변경은 push → 윈도우에서 pull + `pm2 restart` 로 반영한다.

---

## 1. 줄바꿈 (EOL) — 이미 처리됨 ✅

- `.gitattributes` 로 **저장소·체크아웃 모두 LF 통일**(`* text=auto eol=lf`). 맥 clone 시 CRLF 혼입 없음.
- `.bat / .cmd / .ps1` 만 CRLF 유지(윈도우 셸용), 바이너리(zip·이미지·pyc 등)는 변환 금지.
- **맥에서 줄바꿈 신경 쓸 필요 없음.** 에디터 기본(LF) 그대로 쓰면 됨.

---

## 2. .env — 저장소에 없음 (수동 복사 필수)

- `.env` 는 `.gitignore` 대상(시크릿). **clone해도 안 들어옴.**
- 맥에서 실행이 필요하면 `.env.example` 복사 후 키 채우기:
  ```bash
  cp .env.example .env   # 후 값 입력
  ```
- ⚠️ **`.env` 의 절대경로는 윈도우 기준**이라 맥에서 무효:
  - `autostock/config.py` 기본값에 하드코딩됨:
    - `MARKETFLOW_DATA_DIR = D:\INFORUN\chutzrit\autostock\marketflow\data`
    - `MARKETFLOW_SCRIPT = D:\...\marketflow\daily_update.py`
    - `AFTERNOON_SCANNER_SCRIPT = D:\...\engine\run_afternoon.py`
  - 맥에서 이 경로를 쓰는 코드를 돌리려면 `.env` 에서 맥 경로로 override 필요.

---

## 3. 서버 운영은 윈도우 전용 (맥에서 구동 금지)

`main.py` 와 운영 인프라는 윈도우에 묶여 있음:
- 프로세스 관리: `C:\Windows\System32\taskkill.exe` / `tasklist.exe` / `wmic.exe` 직접 호출 → **맥에서 동작 안 함**
- 감독자: **PM2 단독**(`pm2 start ecosystem.config.json` / `pm2 restart autostock`) — 윈도우에 설정됨
- 단일 인스턴스 락(`logs/serve.lock`), `server.pid` 등도 윈도우 운영 전제
- 결론: **맥에서 `python main.py --serve` / PM2 실행하지 말 것.** (충돌·오동작)

> 관련 메모리: 감독자 PM2 단독 / 자체 데몬 봉인.

---

## 4. 맥에서 할 수 있는 것 (권장 용도)

- **코드 편집·리뷰** (Python 3.12 기준)
- **백테스트·연구** (`scratch/backtest_*.py`) — FinanceDataReader·pandas·requests 기반이라 맥에서 잘 돔
- 단위 테스트: `pytest`
- 환경 구성:
  ```bash
  python3 -m venv .venv && source .venv/bin/activate
  pip install -r requirements.txt
  ```
- ⚠️ 백테스트 중 일부는 **Naver 스크래핑(euc-kr)** 사용(`backtest_supply_ic.py`, `backtest_stackC/D.py`) — 맥에서도 동작하나 네트워크/차단에 따라 실패 가능.

---

## 5. Git 워크플로 (양 기기 공통)

- **커밋은 요청 시 한 번에.** 중간 자동 커밋 금지(사용자 규칙).
- `master` = 라이브/배포 브랜치. 검증 보류분은 `strategy-redesign-research` 등 별도 브랜치.
- 반영 흐름:
  ```
  (맥/윈도우) 편집 → push  →  (윈도우) git pull → pm2 restart autostock
  ```
  ※ **맥에서 push해도 서버엔 자동 반영 안 됨** — 윈도우에서 pull + 재시작해야 라이브 적용.
  ※ MarketFlow 스캔은 subprocess라 파일만 갱신되면 다음 16:00 스캔부터 반영(서버 재시작 불필요).
- `.env`·`data/`·`logs/`·`*.zip` 등은 gitignore — 커밋되지 않음.

---

## 6. 인코딩 메모

- 로깅·콘솔은 UTF-8 강제(`PYTHONUTF8=1`, 로거 UTF-8 래퍼). 맥은 UTF-8 기본이라 문제 없음.
- 윈도우 콘솔 cp949 이슈는 윈도우 한정 — 맥에선 해당 없음.
- 일본어 사용 금지(한국어/영어만) — 코드 주석·문서 공통.

---

## 7. 체크리스트 (맥에서 처음 clone 시)

- [ ] `git clone` (`.gitattributes` 자동 적용 → LF)
- [ ] `python3 -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt`
- [ ] (실행 필요 시) `.env.example` → `.env` 복사 후 키 입력, 절대경로는 맥 경로로 수정
- [ ] 편집·백테스트·테스트만 수행, **서버는 윈도우에서**
- [ ] 변경 push → 윈도우에서 pull + `pm2 restart autostock`
