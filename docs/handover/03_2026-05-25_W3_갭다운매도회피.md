## 1. 가설

**가설:**
> 손절폭 자체보다 더 큰 손실을 일으키는 건 **갭다운 시 시장가 매도**다.
> stop_price 9,700원으로 설정했어도 시초가 9,200원으로 갭다운되면 시장가 매도는 9,200원 또는 그 이하에 체결된다.
> 즉 **stop은 환상이고 실제 손실은 -8% 이상**일 수 있다.

**현재 동작:**
- `kis_client.market_sell` = 시장가 (`ord_dvsn="01"`)
- `_sell_with_retry`는 매도 실패만 재시도, 슬리피지는 무방비
- 09:00~09:05 갭다운 종목을 그 자리에서 시장가로 던짐

**기대 효과:**
- 갭다운 종목에 대해 **5분간 회복 대기** → 평균 매도가 +1~2% 개선 기대
- 회복 안 되면 그때 매도 (손실 한도는 동일)

## 2. 작업 단계

### Step 1. 갭다운 매도 버퍼 (월 오전)

**파일:** `autostock/trading/trailing_stop.py`

상수 추가:
```python
MORNING_OPEN_END = dtime(9, 5)            # 09:05까지를 "장 시작 직후"
GAP_DOWN_BUFFER_MIN = 5                   # 갭다운 시 매도 대기 시간 (분)
GAP_DOWN_THRESHOLD_PCT = 2.0              # 진입가 대비 추가 -2% 이상이면 갭다운으로 간주
```

`_sell_with_retry` 호출 직전에 갭다운 가드 추가:

```python
async def _try_sell_with_morning_buffer(pos: TrailingPosition, price: float, reason: str) -> bool:
    """
    09:00~09:05 사이에 stop 트리거가 발생하고
    가격이 stop_price 대비 추가 -2% 이상 하락이면
    5분 대기 후 재평가.
    """
    now = _now_kst()
    is_morning = MARKET_OPEN <= now <= MORNING_OPEN_END
    if not is_morning:
        return await _sell_with_retry(pos, price, reason)

    # 갭다운 강도 측정
    gap_below_stop = (pos.stop_price - price) / pos.stop_price * 100
    if gap_below_stop < GAP_DOWN_THRESHOLD_PCT:
        # 평범한 stop 히트 — 즉시 매도
        return await _sell_with_retry(pos, price, reason)

    # ── 갭다운 감지 — 5분 회복 대기 ──
    log.warning(
        "[%s] 갭다운 감지: price=%.0f stop=%.0f (%.2f%% 아래) — %d분 회복 대기",
        pos.ticker, price, pos.stop_price, gap_below_stop, GAP_DOWN_BUFFER_MIN
    )
    bot_ui.schedule_message(
        f"⚠️ *{pos.name}({pos.ticker})* 갭다운 감지\n"
        f"매도 보류 — {GAP_DOWN_BUFFER_MIN}분 회복 관찰"
    )

    await asyncio.sleep(GAP_DOWN_BUFFER_MIN * 60)

    cur = get_current_price(pos.ticker)
    if cur <= 0:
        log.warning("[%s] 회복 관찰 후 가격 조회 실패 — 즉시 매도", pos.ticker)
        return await _sell_with_retry(pos, price, reason)

    if cur >= pos.stop_price:
        log.info(
            "[%s] 갭다운 회복 — stop 이상 복귀 (%.0f ≥ %.0f) 매도 보류 유지",
            pos.ticker, cur, pos.stop_price
        )
        bot_ui.schedule_message(
            f"🔄 *{pos.name}({pos.ticker})* 갭다운 회복\n"
            f"현재 {cur:,.0f} ≥ stop {pos.stop_price:,.0f} — 보유 유지"
        )
        return False  # 매도 안 함, 다음 폴링에서 다시 평가

    # 회복 실패 — 매도 진행
    log.warning("[%s] 갭다운 회복 실패 — 매도 진행 (cur=%.0f)", pos.ticker, cur)
    return await _sell_with_retry(pos, cur, reason + "(갭다운)")
```

```python
expected = pos.stop_price
slippage = (price - expected) / expected * 100   # 음수면 시장가 손해
emoji = "🔴" if slippage <= -1.0 else "🟠"
bot_ui.schedule_message(
    f"{emoji} *{pos.name}({pos.ticker})* {reason} 완료\n"
    f"예상 stop {expected:,.0f} → 실제 매도 {price:,.0f} (슬리피지 {slippage:+.2f}%)"
)
```
