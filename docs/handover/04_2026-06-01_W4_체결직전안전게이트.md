## 1. 가설

**가설:**
> AI 점수 임계값(scorer 9점, supervisor 5/7점)은 **사용자 룰에 의해 변경 금지**.
> 그러나 점수가 통과해도 **시장 상황(거래량 / 등락률 / 추세)** 이 위험 신호를 보내는 종목은 차단해야 한다.
> 이건 점수 변경이 아니라 **체결 직전 안전망**이다.

**현재 동작:**
- HITL 자동 승인 → execute_decisions → 갭업 가드(W1) → 매수
- **거래량 폭발(RVOL) 없는 종목도 통과** (RVOL은 스코어에만 반영, 게이트 아님)
- **+10% 이상 급등한 종목도 통과** (위치가 너무 높음)
- **하락 추세 (MA20 이탈) 종목도 통과**

**기대 효과:**
- 약신호 종목의 진입 차단으로 손절 후보 제거
- 임계값 건드리지 않고 진입 품질 향상

## 2. 작업 단계

### Step 1. 게이트 모듈 (월 오전)

**파일 (신규):** `autostock/trading/entry_gates.py`

```python
"""체결 직전 안전 게이트 — 점수 임계값 변경 없이 약신호 차단."""
from typing import Optional
import pandas as pd

from autostock.market.fetcher import fetch_ohlcv
from autostock.logger import get_logger

log = get_logger(__name__)


# ── 게이트 임계값 ──────────────────────────────────
# 이 값들은 사용자 룰의 "기준값(임계값)"이 아닌
# 시장 데이터 기반 안전선이므로 조정 가능합니다.
EXTENDED_CHANGE_PCT = 7.0     # 당일 등락률 ≥ +7% 이면 추격매수로 차단
RVOL_MIN_RATIO = 1.3          # 20일 평균 대비 거래량 1.3배 미만이면 차단
MA20_PROXIMITY_PCT = -2.0     # MA20 대비 -2% 이하면 추세 훼손으로 차단


def _vol_ratio(df: pd.DataFrame) -> float:
    if len(df) < 21:
        return 0.0
    vol_20avg = df["Volume"].iloc[-21:-1].mean()
    if vol_20avg <= 0:
        return 0.0
    return float(df["Volume"].iloc[-1] / vol_20avg)


def _ma20_distance_pct(df: pd.DataFrame) -> float:
    if len(df) < 20:
        return 0.0
    ma20 = df["Close"].iloc[-20:].mean()
    cur = df["Close"].iloc[-1]
    if ma20 <= 0:
        return 0.0
    return float((cur / ma20 - 1) * 100)


def check_entry_gates(
    ticker: str,
    change_pct: float,
    cur_price: float,
) -> Optional[str]:
    """
    체결 직전 게이트 검사.
    Returns:
        None: 모든 게이트 통과
        str:  차단 사유 (BLOCK_EXTENDED / BLOCK_RVOL / BLOCK_MA20)
    """
    # 1. 추격매수 차단
    if change_pct >= EXTENDED_CHANGE_PCT:
        log.warning("[%s] 게이트 차단 EXTENDED — change_pct=%+.1f%% >= %.1f%%",
                    ticker, change_pct, EXTENDED_CHANGE_PCT)
        return "BLOCK_EXTENDED"

    df = fetch_ohlcv(ticker, days=30)
    if df is None or df.empty:
        log.warning("[%s] OHLCV 조회 실패 — 게이트 통과 (보수적)", ticker)
        return None

    # 2. 거래량 폭발 확인
    rvol = _vol_ratio(df)
    if 0 < rvol < RVOL_MIN_RATIO:
        log.warning("[%s] 게이트 차단 RVOL — ratio=%.2f < %.2f",
                    ticker, rvol, RVOL_MIN_RATIO)
        return "BLOCK_RVOL"

    # 3. MA20 추세 훼손 확인
    ma20_dist = _ma20_distance_pct(df)
    if ma20_dist <= MA20_PROXIMITY_PCT:
        log.warning("[%s] 게이트 차단 MA20 — 거리 %+.2f%% <= %.1f%%",
                    ticker, ma20_dist, MA20_PROXIMITY_PCT)
        return "BLOCK_MA20"

    log.info("[%s] 게이트 통과: change=%+.1f%% rvol=%.2f ma20_dist=%+.2f%%",
             ticker, change_pct, rvol, ma20_dist)
    return None
```

> ⚠️ **임계값 7%, 1.3배, -2%는 점수 임계값이 아닌 시장 안전선**입니다. 사용자 룰의 "기준값 변경 금지"는 scorer 등급(11/9/9), confidence(5/7) 같은 **모델 출력 컷오프**에 적용됩니다. 안전 게이트의 시장 데이터 임계값은 일반 파라미터로 분류됩니다.
>
> 그래도 처음 적용 시에는 사용자께 한 번 더 확인받으십시오.

## 4. 회고 의사결정 (6/5 금 17:00)

| 항목 | W3 결과 | W4 결과 |
|---|---|---|
| 손절률 | __% | __% |
| 게이트 차단 종목의 다음 5일 수익률 (놓친 기회 측정) | — | 평균 __% |
| 게이트 통과 후 매수의 손절률 | — | __% |

**Go 조건:**
- [ ] 게이트 통과 종목의 손절률이 차단 전 평균보다 낮음
- [ ] 차단된 종목 중 +5% 이상 상승 사례 ≤ 30% (놓친 기회 비율 허용 범위)

**No-Go:**
- 차단된 종목이 자주 큰 수익을 냄 (가드가 너무 보수적) → 임계값 완화 검토

**🔔 임계값 검토 트리거 점검 (W4 회고 시 필수):**

W1~W4 작업 후에도 **손절률 ≥ 30%** 가 지속되면 `09_점수임계값_검토_가이드.md` §3 Trigger A 발동을 사용자께 보고해야 합니다. 이 시점이 시장 데이터 가드만으로는 손절률을 낮출 수 없다는 신호입니다.

판단 기준:
- 손절률 ≥ 30% AND 차단된 종목과 통과 종목의 향후 5일 수익률이 비슷 → **Trigger A 발동 권고**
- 손절률 ≥ 30% AND 차단된 종목이 자주 +10% 이상 수익 → **가드 완화 우선** (임계값 미변경)
- 손절률 < 30% → **임계값 검토 불필요** (W5 정상 진행)
- 매수 표본 < 5건/주 → **Trigger B 발동** (가드 완화)

**Trigger A 발동 시 사용자 보고 양식:**
```

  RVOL/MA20 게이트로 회피된 손실 추정: ___%

남은 손실의 원인 분석:
  - (예: 진입 후 5일 내 -3% 도달 종목들의 confidence 평균 = ___, 5점 부근 약신호 다수)
```
