## 1. 가설

**가설 A — 눌림 대기:**
> 강한 종목은 시초가 직진 후 5MA로 한 번 내려왔다 다시 올라간다.
> 시초가 부근에서 매수하면 직진의 끝(고점)을 사고, 이후 눌림에 손절선이 깨진다.
> 09:15 이후 5분봉 5MA 부근까지 눌림을 기다려 매수하면 평균 진입가 -0.5~1.0% 개선.

**가설 B — 워치리스트 신선도:**
> 어제 16:00 스캔된 신호는 다음 날 09:00 매수까지 17시간 묵는다.
> 그 사이 갭업이 +5% 이상 발생한 종목은 **이미 갈 만큼 갔다**.
> 08:00 시점에 시간외 호가 또는 야간 글로벌 시황으로 워치리스트를 재정렬해야 한다.

### Step 1. 가설 A — 눌림 대기 진입 (월~화)

**파일:** `autostock/trading/limit_order.py`

```python
"""지정가 매수 + 눌림 대기 (09:15 이후 5MA 부근)."""
import asyncio
from datetime import datetime, time as dtime
import pytz
from autostock.trading.kis_client import (
    get_current_price, limit_buy, get_order_fill_qty, cancel_order, round_to_tick,
    get_intraday_5min,    # ⭐ 신규 — 5분봉 조회 (Step 2에서 구현)
)

KST = pytz.timezone("Asia/Seoul")
PULLBACK_WAIT_START = dtime(9, 15)
PULLBACK_TIMEOUT    = dtime(9, 45)
PULLBACK_TOLERANCE_PCT = 0.5    # 5MA 대비 ±0.5% 이내 진입


async def _wait_for_pullback(ticker: str) -> tuple[bool, int]:
    """
    09:15 ~ 09:45 사이 5분봉 5MA 부근까지 눌림 대기.
    Returns:
        (success, target_price): success=False면 추격 포기
    """
    while True:
        now = datetime.now(KST).time()
        if now >= PULLBACK_TIMEOUT:
            return False, 0  # 30분 동안 눌림 없음 → 진입 포기

        if now < PULLBACK_WAIT_START:
            await asyncio.sleep(60)
            continue

        bars = get_intraday_5min(ticker, count=5)
        if not bars or len(bars) < 5:
            await asyncio.sleep(60)
            continue

        ma5 = sum(b["close"] for b in bars) / len(bars)
        cur = get_current_price(ticker)
        gap_pct = abs((cur / ma5 - 1) * 100)

        if gap_pct <= PULLBACK_TOLERANCE_PCT:
            target = round_to_tick(ma5)
            return True, target

        await asyncio.sleep(60)


async def limit_buy_with_pullback(
    ticker: str,
    qty: int,
    price_reference: float = 0.0,
) -> tuple[bool, int]:
    """눌림 대기 후 지정가 매수."""
    success, target = await _wait_for_pullback(ticker)
    if not success:
        log.info("[%s] 눌림 미발생 — 추격매수 포기", ticker)
        return False, 0

    log.info("[%s] 눌림 도달 — 5MA %d 지정가 진입", ticker, target)
    # 기존 limit_buy 로직 1회 시도 (재시도 없음, 눌림은 한 번이면 됨)
    data = limit_buy(ticker, qty, target)
    order_no = data.get("output", {}).get("ODNO", "")
    if not order_no:
        return False, 0

    # 5분 체결 대기
    for _ in range(10):  # 30초 × 10 = 5분
        await asyncio.sleep(30)
        filled = get_order_fill_qty(order_no)
        if filled >= qty:
            return True, target
    cancel_order(order_no, ticker, qty, target)
    return False, 0
```

> ⚠️ **기존 `limit_buy_with_timeout`은 보존**, 신규 함수로 분리. `executor.py`에서 어느 것을 쓸지 환경변수로 토글:
>
> ```python
> USE_PULLBACK_ENTRY = os.environ.get("ENTRY_MODE", "pullback") == "pullback"
> if USE_PULLBACK_ENTRY:
>     filled, fill_price = await limit_buy_with_pullback(d.ticker, qty, d.price_reference)
> else:
>     filled, fill_price = await limit_buy_with_timeout(d.ticker, qty, d.price_reference)
> ```

def get_intraday_5min(ticker: str, count: int = 20) -> list[dict]:
    """
    당일 5분봉 OHLC 조회.
    KIS API: /uapi/domestic-stock/v1/quotations/inquire-time-itemchartprice
    """
    url = f"{_base_url()}/uapi/domestic-stock/v1/quotations/inquire-time-itemchartprice"
    headers = _get_headers("FHKST03010200")
    params = {
        "FID_ETC_CLS_CODE": "",
        "FID_COND_MRKT_DIV_CODE": "J",
        "FID_INPUT_ISCD": ticker,
        "FID_INPUT_HOUR_1": datetime.now(KST).strftime("%H%M%S"),
        "FID_PW_DATA_INCU_YN": "N",
    }
    try:
        r = requests.get(url, headers=headers, params=params, timeout=5)
        items = r.json().get("output2", [])[:count]
        return [{"close": float(it["stck_prpr"])} for it in items]
    except Exception as e:
        log.warning("[%s] 5분봉 조회 실패: %s", ticker, e)
        return []

### Step 3. 가설 B — 08:00 워치리스트 재정렬 (수)

**파일 (신규):** `autostock/scheduler/morning_refresh.py`

```python
"""08:00 KST — 시간외 호가/시간외 종가로 워치리스트 재정렬."""
from autostock.db import supabase as db
from autostock.market.fetcher import fetch_premarket_quote   # ⭐ 신규

WATCHLIST_GAP_FILTER_PCT = 5.0    # 시간외 +5% 이상이면 제외


async def refresh_watchlist_premarket() -> None:
    """
    08:00 실행. 시간외 호가가 어제 종가 +5% 이상인 종목을
    워치리스트에서 임시 제외 (오늘만).
    """
    watchlist = db.fetch_watchlist()
    blocked = []
    for w in watchlist:
        try:
            pre = fetch_premarket_quote(w["ticker"])
            if pre <= 0 or w["prev_close"] <= 0:
                continue
            gap = (pre / w["prev_close"] - 1) * 100
            if gap >= WATCHLIST_GAP_FILTER_PCT:
                blocked.append((w["ticker"], gap))
                db.set_watchlist_skip(w["ticker"], reason=f"premarket_gap_{gap:+.1f}%")
        except Exception as e:
            log.warning("[%s] premarket 조회 실패: %s", w["ticker"], e)
    log.info("프리마켓 갭업 제외 %d종목: %s", len(blocked), blocked)
```

스케줄 등록 (`jobs.py:create_scheduler`):
```python
scheduler.add_job(
    refresh_watchlist_premarket,
    trigger=CronTrigger(day_of_week="mon-fri", hour=8, minute=0, timezone=kst),
    id="morning_refresh",

```

## 7. W6 종료 시점 — 임계값 검토 최종 의사결정

W6 종료 시점에 다음을 한 번 더 점검하십시오.

**시나리오 A: W6 종료 시 손절률 < 20%**
- 시장 데이터 가드만으로 충분히 해결됨. **점수 임계값은 손대지 않음.**
- 사용자께 "W1~W6 작업 종료, 손절률 ___% → ___% 개선" 보고
- 1~2주 안정화 관찰 후 P2/P3 작업 협의

**시나리오 B: W6 종료 시 손절률 20~30%**
- 추가 개선 여지 있음. 가드 임계값 미세 조정 검토 (점수는 미변경).
- 또는 1~2주 추가 관찰 후 재평가.

**시나리오 C: W6 종료 시 손절률 ≥ 30%**
- 시장 데이터 가드의 한계. **`09_점수임계값_검토_가이드.md` Trigger A 발동.**
- Bear 캘리브레이션 정상화 검토를 사용자께 정식 보고.
- 모의 vs 실거래 정책 차등화도 함께 점검 (모의는 공격성 유지 / 실거래는 보수화).

**시나리오 D: W6 종료 시 매수 표본 < 5건/주**
- 가드가 과보수. **Trigger B 발동** — 가드 완화 (점수는 미변경).
- 매수가 과도하게 줄어들면 시스템 무용지물.
