## 0. 후배에게 — 책임자 서문

후배 여러분, 이 인계 문서는 **순서대로** 읽고 적용하라고 만들었습니다. 절대 다음 두 가지를 하지 마십시오.

> ❌ **여러 변경을 한 번에 합치기**
> ❌ **측정 없이 다음 주차로 넘어가기**

지난 한 주 사용자가 모의투자에서 추천 종목을 매수했더니 **대부분 손절을 시도**했습니다. 손절선이 짧아서가 아니라 **진입가가 갭업 자리에 형성되어 정상 변동성에 걷어차이는 구조**였습니다. 즉 손절을 잘 잡은 게 아니라 **손절을 안 당할 자리에서 매수가 일어나지 않는 시스템**이었어요.

이 6주 동안 우리가 할 일은 단 하나 — **"진입의 질을 올린다"**입니다. 손절선은 그 다음입니다.

## 1. 사용자 룰 — 정확한 해석

이 룰은 사용자께서 직접 정한 안전선입니다. **단, 룰을 표면적으로 따르는 게 목적이 아닙니다 — 룰의 정신을 지키는 게 목적**입니다. 사용자께서는 책임자(=관리자)가 근본 원인을 분석하고 결단을 내리는 수정은 허용한다고 명시하셨습니다.

| 룰 | 정확한 의미 | 우리가 지킬 방법 |
|---|---|---|
| **임시방편 임계값 변경 금지** | "결과 맞추기"식 즉흥적 점수 컷오프 조정은 금지. **근본 원인 분석 + 책임자 결단**을 동반한 변경은 허용. | 1차 작업은 시장 데이터 가드(갭/거래량/ATR)로 진행. 임계값 변경 필요 시 `09_점수임계값_검토_가이드.md` 절차를 반드시 거친다. |
| **모의투자 공격적 매수 정책 유지** | KIS_SIMULATED_MODE=true 시 buy_threshold=5는 **데이터 수집 목적의 의도된 공격성**. Bear 점수 기준이 보수적인 것도 동일한 의도. | 점수 컷오프는 그대로. 약신호의 부작용은 갭업/RVOL/MA20/포지션 캡 등 **시장 데이터 안전망**으로 흡수. |
| **HITL 자동승인은 모의 전용** | KIS_SIMULATED_MODE=false 전환 시 반드시 텔레그램 HITL을 거쳐야 함. | 자동승인 분기는 절대 실거래 모드로 확장하지 않는다. 실거래 전환 시 **buy_threshold가 5→7로 자동 강화**되는 것을 보장한다. |
| **한 번에 하나씩 측정** | 변경 1건 → 최소 1주 관찰 → 다음 변경. | 본 6주 일정 자체가 이 룰의 구현. 단 룰 위반이 손절률의 실제 원인으로 측정되면 09 문서 절차에 따라 수정 가능. |

> 💡 **후배에게:** 헷갈리면 "지금 변경하려는 게 *결과를 맞추기 위한 즉흥적 조정*인가, *근본 원인을 발견하고 책임자가 결단한 변경*인가?"를 물어보십시오. 후자라면 09 문서의 절차로 진행 가능합니다. 전자라면 멈추십시오.

## 3. 주간 의사결정 트리 (Go / No-Go)

매주 금요일 17:00 KST 회고 후 다음 4가지 중 하나를 결정합니다.

```
  [측정 결과]
       │
       ▼
   ┌───┴────┐
   │ KPI 개선?│
   └───┬────┘
       │
   ┌───┴───────────────────────────────┐
   │                                   │
   ▼ Yes                               ▼ No
 손절률 ≥ 20% 감소?                    원인 분석
   │                                   │
   ├── Yes  → ✅ 다음 주차 GO            ├── 코드 버그 → 핫픽스 후 1주 연장
   │                                   │
   └── No   → ⚠️ 1주 추가 관찰          ├── 시장 변동 (KOSPI ±5%) → 정상, 진행
                                       │
                                       └── 가설 자체 오류 → 책임자 보고 + 일정 재검토
```

**핵심 KPI (07_KPI대시보드규격.md 참조):**
- 1차: **일일 손절률** = 손절매도건수 / 보유종목수
- 2차: **평균 진입 슬리피지** = (시초가 - 전일종가) / 전일종가 × 100
- 3차: **주간 누적 P&L %**

## 5. 작업 전 공통 체크리스트

각 주차 작업 시작 전 **반드시** 다음을 확인하십시오.

- [ ] 사용자에게 작업 개시 알림 (텔레그램 or 회의)
- [ ] 현재 `held_positions` 보유 종목 백업 (`Supabase` SQL 덤프)
- [ ] `start_server.bat` 또는 `python main.py --daemon`이 정상 동작 중인지 확인
- [ ] 직전 주 KPI 리포트 1부 출력
- [ ] 본인 PC에 `git diff` 또는 ZIP 백업 1부 (롤백 대비)
- [ ] `KIS_SIMULATED_MODE=true` 확인 (실거래 전환 절대 금지)

작업 후:
- [ ] 단위 테스트 통과 (`pytest -m unit`)
- [ ] 모의투자에서 1회 이상 매수→매도 사이클 확인
- [ ] 텔레그램 알림 정상 작동 확인
- [ ] 다음 주 월요일 09:00 자동 실행 전 코드 배포 완료
- [ ] `docs/handover/<주차>_LOG.md` 에 변경 내역 + 첫 결과 기록
