## 1. 가설

**가설:**
> 고정 3% 손절폭은 KOSPI200 대형주(ATRP ~1.5%)에는 너무 넓고,
> KOSDAQ 중소형 급등주(ATRP 4~7%)에는 너무 좁다.
> 종목별 변동성을 무시한 일률 기준은 노이즈 손절을 양산한다.

**근거:**
- 한국 시장 평균 일중 변동폭(ATRP):
  - KOSPI 대형주: 1.0~1.8%
  - KOSPI 중형: 2.0~3.5%
  - KOSDAQ 중소형: 3.5~6.0%
  - 급등주(시그널 종목): 5.0~8.0%
- 현재 3% 고정 손절은 KOSDAQ 종목 대상으로는 **노이즈 안에 손절선이 들어가는** 구조

**기대 효과:**
- 변동성 큰 종목의 노이즈 손절 감소
- 변동성 작은 종목의 손실 한도 더 빠른 차단 (오히려 손실 ↓)

## 2. 작업 단계

### Step 1. ATR 산출 헬퍼 (월 오전)

**파일 (신규 또는 확장):** `autostock/trading/risk.py`

```python
"""변동성 기반 손절폭 계산."""
import pandas as pd
from autostock.market.fetcher import fetch_ohlcv
from autostock.logger import get_logger

log = get_logger(__name__)

ATR_PERIOD = 14
ATR_MULTIPLIER = 1.5
STOP_PCT_FLOOR = 2.5     # 너무 짧지 않게
STOP_PCT_CEIL  = 6.0     # 너무 길지 않게


def compute_atrp(ticker: str) -> float:
    """ATRP(%) = ATR(14) / 종가 × 100. 실패 시 0 반환."""
    try:
        df = fetch_ohlcv(ticker, days=30)
        if df is None or len(df) < ATR_PERIOD + 1:
            return 0.0
        h, l, c = df["High"], df["Low"], df["Close"]
        prev_c = c.shift(1)
        tr = pd.concat([h - l, (h - prev_c).abs(), (l - prev_c).abs()], axis=1).max(axis=1)
        atr = tr.rolling(ATR_PERIOD).mean().iloc[-1]
        close = c.iloc[-1]
        if close <= 0:
            return 0.0
        return float(atr / close * 100)
    except Exception as e:
        log.warning("[%s] ATRP 산출 실패: %s — 기본값 사용", ticker, e)
        return 0.0


def compute_stop_pct(ticker: str) -> float:
    """ATRP × 1.5, [2.5, 6.0] 클램프."""
    atrp = compute_atrp(ticker)
    if atrp <= 0:
        return 3.0  # 산출 실패 시 기존 3% 폴백
    pct = atrp * ATR_MULTIPLIER
    return max(STOP_PCT_FLOOR, min(STOP_PCT_CEIL, round(pct, 2)))
```

`_fixed_stop` 시그니처 변경:

```python
def _fixed_stop(entry: float, stop_pct: float = 3.0) -> int:
    """ATR 기반 손절가 (호가단위 정렬 포함)."""
    from autostock.trading.kis_client import round_to_tick
    return round_to_tick(entry * (1 - stop_pct / 100))


def _trail_stop(peak: float) -> int:
    from autostock.trading.kis_client import round_to_tick
    return round_to_tick(peak * (1 - PROFIT_TRAIL_PCT / 100))
```

> ⭐ **변경 포인트:** `round()` → `round_to_tick()` 으로 동시 교체. (P2-12 항목 W2에 흡수)

from autostock.trading.risk import compute_stop_pct
from autostock.trading.trailing_stop import _fixed_stop

for r in exec_results:

            stop_pct=stop_pct,           # ⭐ 보존

KIS 미등록 자동 추가 분기에도 동일 적용 + **복구 시 phase 결정 로직 보강** (보너스 수정):

```python
# KIS에 있지만 DB에 없는 종목 자동 추가 분기
gain_pct = (cur / entry - 1) * 100 if entry > 0 else 0
stop_pct = compute_stop_pct(ticker)

if gain_pct >= PROFIT_TRIGGER_PCT:
    phase = 'trail'
    stop  = _trail_stop(cur)
    peak  = cur
elif gain_pct >= BREAKEVEN_TRIGGER_PCT:
    phase = 'even'
    stop  = round_to_tick(entry)
    peak  = cur
else:
    phase = 'stop'
    stop  = _fixed_stop(entry, stop_pct)
    peak  = max(entry, cur)
```

> 💡 **버그 수정 보너스:** 기존 코드는 KIS에서 자동 발견한 포지션을 항상 phase='stop'으로 등록해, 이미 +5% 수익 중인 종목도 trail이 활성화되지 않는 문제가 있었습니다. 이번 주에 함께 고칩니다.
